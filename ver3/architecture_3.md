# Архитектура голосового переводчика ver3 (Bergamot WASM)

## Обзор решения

Голосовой переводчик ver3 — браузерное веб-приложение на чистом JavaScript (ES-модули),
работающее без серверной части. Перевод выполняется **локально в браузере** с помощью
нейросетевого движка Bergamot (Mozilla Firefox Translations) через WebAssembly.

---

## Выбор технологии перевода

### Почему Bergamot, а не Argos Translate

| Критерий | Bergamot | Argos Translate |
|----------|----------|-----------------|
| Браузерный JS / WASM | **Да** — официальный WASM-порт | Нет — только Python |
| Работа без сервера | **Да** | Нет |
| GitHub Pages | **Да** (с workaround) | Нет |
| Качество (ru↔en) | Высокое (Marian NMT) | Высокое (OpenNMT) |
| CDN-доступность | **Да** (jsDelivr) | Нет |

Argos Translate существует только в виде Python-библиотеки и требует серверной части
(LibreTranslate). Bergamot имеет официальный WASM-порт, разработанный Mozilla для
встройки в Firefox — именно он используется в функции «Translate Page» в Firefox.

## Компоненты системы

### 1. Модуль конфигурации (`config`)
Объект JS с параметрами:
- `translationDirection` — `"ru-en"` или `"en-ru"`
- `voiceGender` — `"female"` / `"male"`
- `speechRate` — скорость синтеза речи
- `pauseDuration` — пауза (мс) для определения конца блока речи

### 2. WASM-движок перевода Bergamot (`BergamotTranslator`)

**Используемый пакет:** `@browsermt/bergamot-translator@0.4.9`
**CDN:** `https://cdn.jsdelivr.net/npm/@browsermt/bergamot-translator@0.4.9/`

Подкомпоненты:
- **`translator.js`** — JavaScript API-обёртка (~79 КБ)
- **`translator-worker.js`** — Web Worker (~17 КБ), выполняет WASM-код
- **`bergamot-translator-worker.wasm`** — скомпилированный движок Marian NMT (~5 МБ)

Нейросетевые модели (загружаются с Mozilla S3 при первом запуске):

| Направление | Файл модели | Файл лексики | Файл словаря | Суммарно |
|-------------|-------------|--------------|--------------|---------|
| ru→en (ruen) | `model.ruen.intgemm.alphas.bin` ~16 МБ | `lex.50.50.ruen.s2t.bin` ~5 МБ | `vocab.ruen.spm` ~1 МБ | ~22 МБ |
| en→ru (enru) | `model.enru.intgemm.alphas.bin` ~16 МБ | `lex.50.50.enru.s2t.bin` ~3 МБ | `vocab.enru.spm` ~1 МБ | ~20 МБ |

Модели кэшируются браузером — повторные запуски мгновенны.

**Алгоритм перевода (внутри WASM):**
1. Токенизация входного текста с помощью SentencePiece
2. Нейросетевой encoder-decoder (Marian NMT — трансформер)
   - INT8-квантизация весов (INTGEMM) для ускорения вычислений
   - Авторегрессивная генерация токенов целевого языка
   - Beam search (поиск по лучу) для выбора оптимального перевода
3. Детокенизация: токены → текст

### 3. Резервный переводчик (`FallbackTranslator`)
При недоступности Bergamot автоматически используется MyMemory REST API
(как в ver1). Обеспечивает работоспособность при загрузке WASM или на браузерах
без поддержки SharedArrayBuffer.

### 4. Service Worker для COOP/COEP (`coi-serviceworker.js`)
**Проблема:** WASM использует `SharedArrayBuffer`, который требует заголовков:
- `Cross-Origin-Opener-Policy: same-origin`
- `Cross-Origin-Embedder-Policy: require-corp`

GitHub Pages не поддерживает произвольные HTTP-заголовки.

**Решение:** `coi-serviceworker` (Guido Zuidhof, MIT) — Service Worker, который
перехватывает все ответы и добавляет эти заголовки программно. Регистрируется
одним `<script>` тегом в `<head>` страницы. После первой регистрации страница
перезагружается с нужными заголовками.

### 5. Модуль распознавания речи (`SpeechRecognizer`)
Web Speech API — без изменений по сравнению с ver1/ver2:
- Непрерывное распознавание (`continuous: true`)
- Промежуточные результаты для real-time отображения
- Таймер паузы для определения конца блока

### 6. Модуль синтеза речи (`SpeechSynthesizer`)
Web Speech API `SpeechSynthesis` — без изменений по сравнению с ver1/ver2.

## Этапы преобразования (полный цикл)

### Этап 1: Захват звука
- Браузер через WebRTC API захватывает аудиопоток с микрофона
- Обработка ведётся в реальном времени нативным движком браузера

### Этап 2: Распознавание речи (STT)
- `SpeechRecognition` (Google Cloud Speech-to-Text для Chrome/Edge)
- Возвращает промежуточные и финальные транскрипции
- При паузе > `pauseDuration` мс считается, что блок речи завершён

### Этап 3: Нейросетевой перевод (NMT) через Bergamot WASM
1. **Токенизация SentencePiece** — разбивка текста на подслова
2. **Encoder** (трансформер) — кодирует исходный текст в векторное представление
3. **Decoder** (трансформер) — авторегрессивно генерирует токены перевода
4. **Beam search** — выбор лучших гипотез перевода (ширина луча = 4)
5. **Квантизация INT8 (INTGEMM)** — ускорение матричных умножений в 4–8× без GPU
6. **Детокенизация** — обратное преобразование токенов в текст

### Этап 4: Синтез речи (TTS)
- `SpeechSynthesis` с голосом, соответствующим целевому языку и выбранному полу
- Браузерные голоса (OS-level TTS: Microsoft, Apple, Google)

## Ограничения

### Первая загрузка
- Модель для каждого направления (~15–22 МБ) скачивается при первом запуске
- Последующие запуски используют кэш браузера (Cache API / HTTP-кэш)

### Браузерная совместимость
- SpeechRecognition: только Chrome / Edge (Firefox не поддерживает)
- SharedArrayBuffer: требует COOP/COEP заголовков (обеспечивается coi-serviceworker)
- WASM: поддерживается всеми современными браузерами (Chrome 57+, Firefox 52+, Safari 11+)

### Качество перевода
- Bergamot использует те же модели, что Firefox Translations — качество хорошее,
  но ниже GPT-4/DeepL уровня
- Отдельные редкие слова и специализированная терминология могут переводиться неточно

### Service Worker и GitHub Pages
- При первом открытии страницы coi-serviceworker регистрирует SW и перезагружает страницу
- В Firefox кнопка "Старт" будет недоступна (нет SpeechRecognition), но загрузка движка произойдёт

### Производительность
- Перевод на CPU может занимать 0.5–2 с для длинных фраз (зависит от железа)
- INT8 квантизация ускоряет работу, но на мобильных устройствах может быть медленнее
